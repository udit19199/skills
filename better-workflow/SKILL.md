# Better Workflow

Make work visible through outcomes.

## Core Loop

1. State the outcome before the activity.
2. Identify the artifact that will prove progress: PR, issue, doc, preview, demo, log, metric, or decision.
3. Surface blockers with evidence: what failed, what was tried, and what decision or access is needed.
4. Close the loop at EOD with completed outcomes and links.

Completion: the update names outcomes, artifacts, blockers, and verification. If any part is missing, ask for it or mark it unknown.

## Status Shape

Use this format for daily updates:

```text
Plan:
- <deliverable>, <issue/PR/doc link>, and <intended outcome>.

Problems:
- <blocker>, <what I tried>, and <decision/access needed>.

Progress:
- <completed outcome>, <artifact link>, and <verification evidence>.
```

Prefer "Ship provider credential validation tests in PR #123" over "Work on tests".

## Ask For Help

Never send a vague "I am stuck" message. Include:

- Goal
- Current state
- Exact failure or uncertainty
- Files, PR, logs, screenshots, or command output
- Attempts already made
- The specific decision or review needed

## Collaboration Tools

Use available connectors when they exist:

- GitHub for issues, PRs, reviews, CI, and code context.
- Jira or Atlassian tools for sprint tickets and product defects.
- Teams or Slack for project-channel updates.
- SharePoint or Google Drive for PRDs, sprint review docs, and shared artifacts.
- Calendar for 1:1s, sprint planning, exits, retros, and meeting prep.

If a connector is unavailable, produce a ready-to-send message or checklist instead of pretending to have posted it.

## Voice

Write like an engineer-operator:

- Short paragraphs.
- Concrete nouns.
- Clear tradeoffs.
- No hype, no filler, no performative positivity.
